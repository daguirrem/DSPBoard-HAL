◆ spi_master_init()

void spi_master_init ( spi_t *  spi)

Init SPI as master.

spi:master:initialization

Configure a target .spi_p from /spi as master with /spi configurations. Also configure a respective GPIO to work with the peripheral.

Parameters
[in]spiAddress to spi variable configurator.
Returns
None.
Note
1. .sck_pin, .miso_pin & .mosi_pin from /spi must be from same GPIO configured at .gpio_spi.
2. .ss_pin is an array of gpio_io_t, and .ss_size must have a .ss_pin array size
See also

E.g.

...
// This will be inicializate SPI2 as master with:
// - SCK, MISO & MOSI = 15, 14, 13 GPIO pinnes from GPIOB
// - SS = 10 & 9 GPIO pinnes from GPIOC (2 Slaves) controlled by software.
// - Clock prescaler = 256
// - 8 Bits MSB first.
// - SPI standar mode = 3.
int main(void)
{
spi_clk_en(SPI2_RCC);
spi_t *spi = {
.spi_p = SPI2, .gpio_spi = GPIOB, .gpio_ss = GPIOC,
.sck_pin = GPIO_IO15, .miso_pin = GPIO_IO14, .mosi_IO13,
.ss_pin = {GPIO_IO9, GPIO_IO10}, .ss_size = 2,
.software_ss = true,
.clk_prescaler = SPI_PRESCALER_256,
.data_format = SPI_DF_8BITS,
.transfer_format = SPI_TF_MSB,
.mode = SPI_MODE_3
}
spi_master_init(&spi);
}

Execute a write comand to target spi on target slave.

spi:master:write

Writes /data to DR register on .spi_p from /spi with control of target SS signal selected by /ss (indexing .ss_pin array from /spi) by software or hardware depending of .software_ss enabler on /spi. The data format transaction will be 8 or 16 bits depending .data_format on /spi.

Parameters
[in]spiTarget SPI configuration to write.
[in]dataTarget data to write.
Returns
None.
Note
Hardware SS control not implemented yet.
Todo:
Hardware SS control.

E.g.

...
// Note: By default spi mode = 0, clk_prescaler = SPI_PRESCALER_2,
// .transfer_format = SPI_TF_MSB
int main(void)
{
spi_clk_en(SPI2_RCC);
spi_t *spi = {
.spi_p = SPI2, .gpio_spi = GPIOB, .gpio_ss = GPIOC,
.sck_pin = GPIO_IO15, .miso_pin = GPIO_IO14, .mosi_IO13,
.ss_pin = {GPIO_IO9, GPIO_IO10}, .ss_size = 2
.data_format = SPI_DF_8BITS
}
spi_master_init(&spi);
// Write 0xA5 to SS @ GPIO_IO9
spi_master_write(&spi, 0xA5, 0);
// Write 0xA6 to SS @ GPIO_IO10
spi_master_write(&spi, 0xA6, 1);
while(1);
}

Execute a write and read comand to target spi on target slave.

spi:master:write:and:read

Writes /data to TX DR register on .spi_p from /spi with control of target SS signal selected by /ss (indexing .ss_pin array from /spi) by software or hardware depending of .software_ss enabler on /spi. And returns RX DR register from target spi. The data format transaction will be 8 or 16 bits depending .data_format on /spi.

Parameters
[in]spiTarget SPI configuration to write and read.
[in]dataTarget data to write.
Returns
(uint16_t) Target SPI data readed.
Note
Hardware SS control not implemented yet.
Todo:
Hardware SS control.

E.g.

...
// Note: By default spi mode = 0, clk_prescaler = SPI_PRESCALER_2,
// .transfer_format = SPI_TF_MSB
int main(void)
{
// SPI data inputs.
uint8_t spi_in_0, spi_in_1;
spi_clk_en(SPI2_RCC);
spi_t *spi = {
.spi_p = SPI2, .gpio_spi = GPIOB, .gpio_ss = GPIOC,
.sck_pin = GPIO_IO15, .miso_pin = GPIO_IO14, .mosi_IO13,
.ss_pin = {GPIO_IO9, GPIO_IO10}, .ss_size = 2
.data_format = SPI_DF_8BITS
}
spi_master_init(&spi);
// Write 0xA5 to SS @ GPIO_IO9 (Expect an answer)
spi_in_0 = (uint8_t) spi_master_write(&spi, 0xA5, 0);
// Write 0xA6 to SS @ GPIO_IO10 (Expect an answer)
spi_in_1 = (uint8_t) spi_master_write(&spi, 0xA6, 1);

void spi_rst ( spi_rcc_t  mask)

Reset single or multiple SPI peripheral/s.

spi:reset

Reset masked peripheral/s in RCC_APB2RSTR or RCC_APB1RSTR, This will reset ALL registers associated with the target peripheral/s.

Parameters
[in]maskSPI mask to be reset.
Returns
None.
Note
You can reset multiple SPI's at same time passing each of them with an OR operator.
See also
spi_rcc_t Mask type.
rcc_rst_apb1 Internal RCC function.
rcc_rst_apb2 Internal RCC function.

E.g.
For one SPI peripheral reset:

...
// This reset SPI 1 peripheral
spi_rst(SPI1_RCC);
...

For multiple SPI peripheral reset:

...
// This reset SPI 2 and SPI 1 peripherals
spi_rst(SPI2_RCC | SPI1_RCC);
